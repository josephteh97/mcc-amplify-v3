================================================================================
  LM STUDIO SETUP & BENCHMARK WORKFLOW
  For VLM floor plan column detection testing
================================================================================


STEP 1 — ADD lms TO PATH (one-time, run in terminal)
-----------------------------------------------------
  echo 'export PATH="$HOME/.lmstudio/bin:$PATH"' >> ~/.bashrc && source ~/.bashrc

  Verify:
    lms --version


STEP 2 — START THE SERVER
--------------------------
  lms server start

  Verify it's running on port 1234:
    lms server status

  Stop when done:
    lms server stop


STEP 3 — PER-MODEL TEST CYCLE (repeat for each of the 12 models)
-----------------------------------------------------------------
  a) Download the model (first time only):
       lms get <model-id>

  b) Load into VRAM:
       lms load <model-id>

  c) Run the benchmark:
       cd ~/Documents/mcc-amplify-v2/pdf_detection_agent
       conda run -n yolo python benchmark/lmstudio_benchmarker.py

  d) Unload from VRAM:
       lms unload <model-id>

  e) Repeat from (a) with next model.


STEP 4 — VIEW ACCUMULATED RESULTS AT ANY TIME
----------------------------------------------
  conda run -n yolo python benchmark/lmstudio_benchmarker.py --results

================================================================================
  BENCHMARK SCRIPT DETAILS
================================================================================

  Script  : benchmark/lmstudio_benchmarker.py
  Results : benchmark/lmstudio_results.json   (appended after each test)
  Log     : benchmark/lmstudio_results.log

  The script auto-detects whichever model is currently loaded in LM Studio.
  If multiple models are loaded, pass --model "model-id" to specify.

  Extra options:
    --image /path/to/tile.png    Use a different test image
    --url http://host:port/v1    If LM Studio is on a different port
    --results                    Print summary table and exit

================================================================================
  TROUBLESHOOTING
================================================================================

  "Cannot reach LM Studio"
    → Run: lms server start
    → Check: lms server status  (should show port 1234)

  "No model loaded"
    → Run: lms load <model-id>
    → Check: lms ps  (lists models currently in memory)

  "lms: command not found"
    → Run: source ~/.bashrc   (or open a new terminal)
    → Or use full path: ~/.lmstudio/bin/lms

  Model not found by lms get
    → Try partial name search: lms get qwen   (shows matching results)
    → Or download via LM Studio desktop app, then use lms ls to find the ID
