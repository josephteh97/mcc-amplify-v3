================================================================================
  ROOT CAUSE — WHY ALL LOCAL MODELS FAIL
================================================================================

  The core issue is localization precision, not visual understanding.

  1. COLUMN SIZE vs IMAGE SIZE
     True column symbols are 40-50 px in a 1285x1273 px page image.
     After resizing to 1024 px for the model, each column is ~32-40 px wide
     (~3% of image width). General-purpose VLMs are not trained to localize
     such small objects with sub-pixel precision.

  2. COARSE vs TIGHT BOUNDING BOXES
     Models output 100-300 px approximate boxes around structural regions.
     Ground truth boxes are 40-50 px tight around the actual column symbol.
     Overlap (IoU) between coarse model boxes and tight GT boxes is effectively
     zero — even if the model is "looking at the right area."

  3. DOMAIN GAP
     VLMs are trained on natural images and general documents. They understand
     that this is a floor plan, but cannot distinguish between column symbols,
     grid balloons, wall intersections, and door arcs at the precision required.

  4. NO FINE-TUNING
     All models tested are zero-shot (no floor plan training examples). Without
     fine-tuning on architectural drawings, even the 11B LLaMA model fails.

  EDGE DEVICE VERDICT
================================================================================

  ANSWER: NO — Do not buy an edge device for VLM-based detection.
          YES — Buy ONLY if deploying YOLO on-site.

  DETAILED REASONING:

  1. VLMs CANNOT DO THIS TASK ON EDGE
     This entire benchmark (31 models, 14 hours of testing) proved that no local
     vision model — not even on a workstation GPU — can reliably localize 40-50 px
     column symbols in architectural floor plans. Every model scored 0.000 P/R/F1.
     Running the same failing models on weaker edge hardware will not fix this.
     A Jetson cannot run what a Quadro RTX 4000 already fails at.

  2. VLMs ARE TOO SLOW ON EDGE FOR YOUR PIPELINE
     Even if a VLM could detect columns, at 1-5 tok/s on Jetson it takes
     40 minutes to 2.6 hours per PDF. Your pipeline needs < 30 seconds.

  3. YOLO IS THE CORRECT EDGE SOLUTION
     Your column-detect.pt runs at 100+ FPS on Jetson via TensorRT.
     A full 35-tile floor plan completes in < 1 second. This is the only
     viable approach for on-site edge deployment.

  4. IF YOU NEED ON-SITE DEPLOYMENT:
     Buy: Jetson Orin Nano Super ($249 / ~$350 SGD)
     Why: Your YOLO model is only 5.6 MB — the Nano is massively overpowered
          for this. 67 TOPS and 1024 CUDA cores for a 5.6 MB model is overkill.
          No need for Orin NX ($599) or AGX Orin ($1,999).
     How: Convert YOLO to TensorRT, deploy with a simple FastAPI server on Jetson.
          Grid detection (SEA-LION) stays on the cloud/workstation.
          Revit API stays on Windows.

  5. IF YOU NEED BETTER VLM ACCURACY:
     Use OpenAI GPT-4o via API ($0.15/PDF). It is 1000x better than any local
     model at this task, and costs less than the electricity to run a Jetson
     for the same number of PDFs.

  6. DO NOT BUY AGX ORIN ($1,999) FOR THIS PROJECT
     Even with 64 GB RAM and 275 TOPS, the VLM accuracy problem remains.
     The bottleneck is model capability (zero-shot VLMs cannot localize small
     CAD symbols), not compute power. Spending $2,000 on hardware will not
     fix a $0 model architecture problem.
