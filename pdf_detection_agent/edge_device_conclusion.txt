VERDICT: NO — you do NOT need to buy an edge device right now.
────────────────────────────────────────────────────────────────────────────────
The benchmark proved the bottleneck is MODEL CAPABILITY, not hardware.
Buying more VRAM does not fix the root cause.


WHY EDGE DEVICES WON'T HELP (the real problem)
────────────────────────────────────────────────
After testing 14 Ollama vision models (from 1.7 GB to 7.9 GB), every single
model scored 0 P/R/F1 for column localization. The reason is NOT that the
hardware was too slow or had too little VRAM — it is that:

  1. Column symbols are ~40 px in a 1285 px image (3% of width).
     No general-purpose VLM, at any size, can localize objects that small
     with IoU >= 0.25 precision without fine-tuning on floor plan data.

  2. The best model tested (llama3.2-vision:11b, 7.9 GB) still returned wrong
     bounding boxes when given enough time to run. Giving it a GPU with 32 GB
     VRAM would make it run faster — but it would still output wrong coordinates.

  3. Floor plan column detection is a domain-specific task. Throwing bigger
     hardware at a general-purpose model does not make it domain-aware.

WHAT ACTUALLY WORKS (your current architecture is correct)
────────────────────────────────────────────────────────────
  ✓ YOLO fine-tuned on floor plans    ~50 ms/page    runs on existing Quadro RTX 4000
  ✓ GPT-4o via API (few-shot)         ~$0.15/PDF     runs via internet, no local GPU needed
  ✗ Any local Ollama VLM              P=R=F1=0       no hardware upgrade changes this


WHEN WOULD AN EDGE DEVICE MAKE SENSE?
────────────────────────────────────────
Scenario 1: On-site deployment at construction sites with no internet
  → GPT-4o is unavailable; YOLO already works offline; only VLM chat is affected
  → The chat feature uses llama3.1:8b (already local), not GPT-4o
  → Current hardware handles everything offline EXCEPT GPT-4o column detection
  → YOLO handles column detection offline just fine

Scenario 2: You want to run a local 70B-class model for the Revit AI chat
  → llama3.1:8b (current) is adequate for BIM chat assistant tasks
  → Upgrading to 70B would improve answer quality but is not critical
  → This would require 32–48 GB VRAM (Mac Mini M4 Pro or RTX 4090)

Scenario 3: You want to test gemma4 / future foundation models locally
  → Wait for Ollama upgrade first (free), then test gemma4:e2b (2.3 GB)
  → If a future model passes the P/R threshold, then revisit hardware
