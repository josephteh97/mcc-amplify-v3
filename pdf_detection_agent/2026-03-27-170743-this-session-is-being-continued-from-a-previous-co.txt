  ---
  How to save corrections to memory — the correct workflow:

  There are three correction types, all stored as normalized 0–1 bboxes in
  memory.json + SQLite:

  ┌─────────┬─────────────────────┬─────────────────────────────────────────┐
  │ Action  │        When         │                   How                   │
  ├─────────┼─────────────────────┼─────────────────────────────────────────┤
  │ confirm │ Detection is real,  │ Select the row → click P1 on image →    │
  │         │ but bbox is wrong   │ click P2 → auto-saved immediately       │
  ├─────────┼─────────────────────┼─────────────────────────────────────────┤
  │         │ Detection is real,  │ Select the row → drag sliders if needed │
  │ confirm │ bbox is fine        │  → Judgement = "correct" → 💾 Save      │
  │         │                     │ Correction                              │
  ├─────────┼─────────────────────┼─────────────────────────────────────────┤
  │ reject  │ Detection is a      │ Select the row → Judgement = "wrong" →  │
  │         │ false positive      │ 💾 Save Correction                      │
  ├─────────┼─────────────────────┼─────────────────────────────────────────┤
  │ add     │ Column was missed   │ Click image (no row selected) → P1 → P2 │
  │         │ entirely            │  → set Shape → ➕ Save New Detection    │
  └─────────┴─────────────────────┴─────────────────────────────────────────┘

  All three paths call agent.add_correction(tile_hash, action, shape, 
  bbox_norm). The bbox is always stored as normalized 0–1 floats (bbox_tile ÷
  tile_info.width/height).

  On the next detection run, _get_tile_context(tile_hash) reads these
  corrections and injects them into the moondream prompt as: REAL columns: 
  square@[0.12,0.34,0.18,0.41] | FALSE positives to skip: 
  round@[0.5,0.5,0.55,0.6]. Last action wins if you save the same bbox twice
  (contradiction resolution).
