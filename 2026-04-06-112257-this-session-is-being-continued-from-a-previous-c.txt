  Question 2: YOLO Misdetection вҖ” the Cross-Element Validator

  This is a real failure mode. At 300 DPI, a thick wall junction looks identical
   to a small column. You need a cross-agent validation layer that runs after 
  all YOLO agents complete, when you have the full picture:

  Stage 1a вҖ” YOLO Column Agent    в”Җв”җ
  Stage 1b вҖ” YOLO Wall Agent      в”Җв”ӨвҶ’ ALL detections pooled вҶ’
  CrossElementValidator
  Stage 1c вҖ” YOLO Beam Agent      в”Җв”ҳ

  The validator applies four checks:

  Check 1 вҖ” Grid Intersection Plausibility
  Columns MUST be near a grid intersection (within ~1.5 grid spacings)
  Any "column" detection far from all grid intersections вҶ’ suspicious

  score += distance_penalty(col_centre, nearest_grid_intersection)

  Check 2 вҖ” Overlap Conflict
  A "column" bbox that overlaps significantly with a detected "wall" bbox
  is likely a wall section, not a column вҖ” flag it

  if iou(col_bbox, wall_bbox) > 0.4:
      flag col as CONFLICT, flag wall as CONFLICT
      вҶ’ human review, not automatic rejection

  Check 3 вҖ” Geometric Plausibility per Element Type
  Column:  min(w,h)/max(w,h) should be вүҘ 0.60 (squareness)
  Wall:    max(w,h)/min(w,h) should be вүҘ 3.0   (elongation)
  Beam:    similar to wall but in span direction

  A "column" with aspect ratio 0.2 is almost certainly a misclassified wall

  Check 4 вҖ” Neighbourhood Consensus
  If an isolated "column" has no other columns within 3Г— the typical
  grid bay width, it is an outlier вҖ” likely a false positive
  (columns appear in grids, not in isolation)

  ---
  The Quarantine System вҖ” Non-Blocking Review

  The critical design principle: never fail the pipeline. Instead, quarantine
  suspicious detections and continue:

  {
      "status":       "quarantined",   # not rejected
      "reason":       "overlap_conflict:wall_023",
      "plausibility": 0.31,
      "resolution":   "human_review",  # flagged in frontend EditPanel
      # original detection fields preserved
  }

  The pipeline continues with high-confidence detections. Quarantined items
  appear in the EditPanel with a warning colour вҖ” the user can confirm,
  reclassify, or dismiss them before the rebuild.
