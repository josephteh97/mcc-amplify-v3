вқҜ # Claude Code Prompt: Agentic Grid Line Detector for Construction Floor Plans 
                                                                                
## Context & Goal                                                               
                                                                                
You are building an **agentic workflow** that processes PDF construction floor  
plans and detects grid lines the way a human would вҖ” by looking at the drawing, 
 reading the labels, and counting the lines.                                    
                                                                                
There is **no algorithm to implement upfront**. There is **no ground truth      
dataset**. The agent receives a PDF, renders it as an image, and uses SEA-LION  
(local Ollama vision model) to visually understand the drawing and return the   
grid line count and their label text.                                           
                                                                                
The reference notebook `~/Documents/test-grid-detector/final_success.ipynb` is  
a **fallback reference**. The agent should first attempt to detect grid lines   
purely through visual reasoning. If it gets stuck, produces low-confidence      
results, or cannot make sense of a drawing, it may read the notebook to         
understand how a working approach solved the same problem and use that as a     
hint to guide its next attempt.                                                 
                                                                                
This is a **sub-project** inside a larger monorepo:                             
- Append new dependencies to **`../requirements.txt`** вҖ” do not create a local  
one                                                                             
- Add the sub-project section to **`../README.md`** вҖ” do not create a local     
README.md                                                                       
- Do not remove or overwrite any existing lines in either file                  
                                                                                
---                                                                             
                                                                                
## Step 0: Read the Reference Notebook (On-Demand, Not Upfront)                 
                                                                                
The agent should **not** read the notebook before trying. It reads it only when 
 needed вҖ” when confidence is persistently low or the drawing is confusing.      
                                                                                
To make the notebook available to the agent at runtime, load its content as     
readable text that can be passed to SEA-LION as context:                        
                                                                                
```bash                                                                         
jupyter nbconvert --to script                                                   
~/Documents/test-grid-detector/final_success.ipynb --stdout >                   
/tmp/reference_approach.py                                                      
```                                                                             
                                                                                
In `agent.py`, implement a `tool_read_reference()` function that:               
- Reads `/tmp/reference_approach.py`                                            
- Extracts the meaningful logic sections (strips boilerplate, imports, display  
calls)                                                                          
- Returns a concise summary string the agent can include in its next SEA-LION   
prompt as a hint                                                                
                                                                                
The agent passes this hint to SEA-LION like: *"A previous approach that worked  
on a similar floor plan did the following: {hint}. Use this as guidance and try 
 again."*                                                                       
                                                                                
This way the notebook is a **hint of last resort**, not a blueprint the agent   
blindly follows.

## Key Constraints                                                              
                                                                                
1. **Vision first, algorithm never** вҖ” SEA-LION looks at the image and reads it 
 like a human. No Hough transforms, no thresholding, no hardcoded detection     
rules.                                                                          
2. **Reference notebook is a fallback, not a blueprint** вҖ” the agent only       
consults `final_success.ipynb` when it cannot reach confident results on its    
own. It uses the notebook as a hint passed to SEA-LION, not as code to execute. 
3. **No ground truth dataset** вҖ” the notebook shows what was detected; the user 
 judges it visually                                                             
4. **SEA-LION is the only detector** вҖ” it is not a validator on top of another  
method                                                                          
5. **Local only** вҖ” all inference via `http://localhost:11434`, no external API 
 calls                                                                          
6. **5 files, flat structure** вҖ” `main.py`, `agent.py`, `pdf_renderer.py`,      
`ollama_client.py`, `test_grid_detection.ipynb`                                 
7. **Dependencies вҶ’ `../requirements.txt`**, README вҶ’ `../README.md`; no local  
copies                                                                          
8. **Confirm SEA-LION model tag** with `ollama list` before hardcoding anywhere 
9. **Notebook improvement areas must be dynamic** вҖ” generated from actual       
results, not hardcoded                                                          
10. **`used_reference_hint: true`** must appear in verbose output whenever the  
fallback was triggered, so the user knows the agent needed a hint
